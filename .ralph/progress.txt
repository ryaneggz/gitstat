## Codebase Patterns
- The `dev:agent` script in `gitstat-app/package.json` sets `NEXTAUTH_URL=http://localhost:3080` inline before `next dev -p 3080` — this is how port overrides work for NextAuth
- NextAuth redirect behavior is controlled by the `NEXTAUTH_URL` env var; it must match the dev server port
- JWT decryption errors on first load are stale cookie artifacts, not real issues
- `NEXTAUTH_SECRET` MUST be set for server-side `getServerSession()` to work — without it, JWT sessions can't be decrypted server-side even though client-side session fetch works
- The `dev:agent` script uses `${NEXTAUTH_SECRET:-dev-agent-secret}` to provide a fallback secret for headless agent workflows
- When testing auth in browser, always clear cookies first to avoid stale JWT tokens from previous server instances
- The dashboard layout (`src/app/dashboard/layout.tsx`) uses server-side `getServerSession()` and redirects to `/login` if null
